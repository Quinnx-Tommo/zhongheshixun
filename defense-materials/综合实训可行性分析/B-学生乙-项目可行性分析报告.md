## 学生乙模块概览

| 维度 | 内容 |
|------|------|
| **业务定位** | 业务主体——课程元信息 + 学员学习行为闭环 |

### 学生乙业务域速记

> **学生乙是「业务主体」**——课程元信息（CRUD）+ 章节编排（药柜式组合）+ 报名/学习进度。Web-student 是乙的客户端。

### 4 张表结构速记

| 表名 | 核心字段 | 角色 |
|------|----------|------|
| `course` | id, name, cover, teacher_id, category, hours, deleted | 课程主表（被 exam/plan/enroll 引用） |
| `course_chapter` | id, course_id, title, content_url, sort, duration | 课程章节（药柜式组合） |
| `course_enroll` | id, user_id, course_id, enroll_time, deleted | 学员报名 |
| `study_record` | id, user_id, chapter_id, progress, last_position, study_time | 学习进度（断点续传） |

## 第四章 同类项目技术方案对比表

### 4.1 三种可替代课程存储方案

#### 方案 A：药柜式组合 1:N（**乙最终选型**）

- **架构**：course 1:N course_chapter
- **优势**：灵活组合视频/PDF/文本；支持章节级别进度；学习时长精细统计
- **劣势**：模型略复杂，需要维护两表

#### 方案 B：单课程 + content 字段

- **架构**：course 1:1 content（JSON 字段存章节数组）
- **优势**：模型简单，单表操作
- **劣势**：扩展性差；视频/PDF/文本混排受限；进度统计难

#### 方案 C：课程包 + 多课程

- **架构**：course_package N:M course
- **优势**：套餐式销售
- **劣势**：与本项目培训计划场景不匹配

### 4.2 《乙模块同类技术方案对比表》

| 对比维度 | 方案 A：药柜式 1:N（乙选型） | 方案 B：单课程 + content 字段 | 方案 C：课程包 + 多课程 |
|----------|------------------------------|--------------------------------|------------------------|
| **架构成熟度** | ⭐⭐⭐⭐⭐ 主流 | ⭐⭐⭐ | ⭐⭐⭐ |
| **章节进度** | ✔ 精确 | ⚠ 粗粒度 | ⚠ 课程级 |
| **多媒体混排** | ✔ 完美 | ⚠ 受限 | ✔ 完美 |
| **扩展性** | 高 | 低 | 中 |
| **复杂度** | 中（两表） | 低（单表） | 高（关联表） |
| **学习时长统计** | ✔ 精确 | ⚠ 粗略 | ⚠ 课程级 |
| **断点续传** | ✔ 章节级 | ⚠ 课程级 | ⚠ 课程级 |
| **报表统计** | ✔ 灵活 | ⚠ 受限 | ⚠ 受限 |
| **毕设适配** | ✔ 可讲解 | ✔ 简单 | ⚠ 复杂 |
| **代表案例** | 慕课网 / 学堂在线 | 部分内部培训 | 极客时间 |

### 4.3 选型决策（乙）

> **结论**：选 **方案 A（药柜式 1:N）**，理由：
> 1. 4 个标杆平台（好医生/华医网/MOOC/智慧树）均采用类似结构
> 2. 章节级断点续传是基层弱网环境的刚需
> 3. 章节级进度可支撑「培训计划完成度」统计
> 4. 报表统计灵活（按章节/课时/进度多维度）
> 5. 数据库已采用该结构（4 张表已设计）

## 第七章 数据安全初步规划

### 7.1 乙模块数据分级

| 数据级别 | 类别 | 保护策略 |
|----------|------|----------|
| **L3 中敏感** | 学习时长、最后位置、进度 | 加密传输 + 权限隔离 |
| **L2 低敏感** | 课程元信息、章节内容 | 访问控制 + 完整性校验 |
| **L1 公开** | 课程简介、目录 | 公开访问 |

### 7.2 安全防护体系（乙模块）

```
┌──────────────────────────────────────────────┐
│              应用安全层（乙）                  │
│  • JWT 鉴权（甲的 token，userId 来自甲）       │
│  • SQL 注入防护（MyBatis-Plus 参数化）         │
│  • 课程/章节权限校验（admin/admin-api/web-api）│
└──────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────┐
│              传输安全层                        │
│  • HTTPS（HLS over HTTPS）                    │
│  • 视频签名 URL（防盗链）                      │
└──────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────┐
│              存储安全层                        │
│  • 视频/课件 OSS 私有 bucket                  │
│  • 进度数据按 user_id 隔离                     │
│  • 软删字段 deleted                           │
└──────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────┐
│              运维安全层                        │
│  • CDN 日志审计                               │
│  • 视频流量监控                                │
└──────────────────────────────────────────────┘
```

### 7.3 合规对照

| 法规 | 要求 | 乙模块落实 |
|------|------|------------|
| 《数据安全法》 | 数据分类分级保护 | ✔ §7.1 |
| 《个人信息保护法》 | 学习行为数据最小必要 | ✔ 仅存进度/时长 |
| 医疗课件版权 | 授权使用 | ✔ 四川省卫健委授权 |

### 7.4 数据安全风险与缓解

| 风险 | 等级 | 缓解 |
|------|------|------|
| 视频盗链 | 中 | 签名 URL + 防盗链 |
| 学习数据泄露 | 低 | 字段不存敏感信息 |
| 课程误删 | 低 | 软删 + 审计日志 |
| 视频存储丢失 | 中 | OSS 多区域备份 |
