# 分工协调 — 4 人接口与数据依赖关系

> **目标**：这张图讲清楚 4 个人**怎么通过接口 + 数据协作**，而不是各自为战。
> 学生甲是**认证中心**，学生乙是**业务主体**，学生丙是**算法核心**，学生丁是**交互门面**。

## 三、4 人数据库表归属与共享表

### 3.1 表归属总表

| 学生 | 专属表（自己主控） | 共享表（只查询） |
|------|-------------------|------------------|
| **甲** | sys_user / teacher / train_plan / plan_course | — |
| **乙** | course / course_chapter / course_enroll / study_record | teacher（读 teacher.name 展示） |
| **丙** | exam / exam_paper / exam_record / exam_answer / question / knowledge_point | course（关联 exam.course_id） |
| **丁** | consult_record / resource_file | sys_user（关联 consult.user_id） |

### 3.2 共享表关系图

```
甲.sys_user.id ──── (1:N) ← 乙.course_enroll.user_id
                  ──── (1:N) ← 乙.study_record.user_id
                  ──── (1:N) ← 丙.exam_paper.user_id
                  ──── (1:N) ← 丙.exam_answer → question ← 丙专属
                  ──── (1:N) ← 丁.consult_record.user_id

甲.teacher.id ──── (1:N) ← ← 乙.course.teacher_id

乙.course.id ──── (1:N) ← ← 丙.exam.course_id
              ──── (1:N) ← ← 乙.course_chapter.course_id

甲.train_plan.id ── (1:N) ← ← 甲.plan_course.plan_id
乙.course.id ──── (1:N) ← ← 甲.plan_course.course_id
```

## 四、4 人跨模块调用接口速查

### 甲调乙

| 场景 | 甲调乙 | 结果 |
|------|--------|------|
| 培训计划关联课程 | `GET /admin/course/list` | 获取课程下拉 |
| 仪表盘概览 | `GET /admin/stats/overview` | 展示总数 |

### 乙调丙

| 场景 | 乙调丙 | 结果 |
|------|--------|------|
| 课程详情显示考试 | `GET /admin/exam/list?course_id=1` | 列出关联考试 |
| 学员考试进度 | `GET /api/exam/record?user_id=xxx` | 查考试记录 |

### 丙调乙

| 场景 | 丙调乙 | 结果 |
|------|--------|------|
| 开始考试需课程信息 | `GET /api/course/detail/{id}` | 拿课程元信息 |
| 成绩统计按课程 | `GET /api/course/list` | 成绩按课程分组 |

### 丁调乙/丙

| 场景 | 丁调 | 结果 |
|------|------|------|
| 咨询被问"课程 1 第 2 章怎么学" | 乙 `/api/course/detail/1` | 章节元信息 |
| 咨询被问"上次考试难度" | 丙 `/api/exam/records` | 考试成绩 |
| 学员问"某道题为什么错" | 丙 `/api/exam/record/${id}` | 看 exam_answer 错题 |
| 小程序课程列表 | 乙 `/api/course/list` | 显示课程 |
| 小程序课程详情 | 乙 `/api/course/detail/{id}` | 章节列表 |
| 小程序开始/交卷考试 | 丙 `/api/exam/start/{id}` `/submit` | 走完整考试流 |
| 小程序培训计划 | 甲 → 乙 → 丁调甲 TrainPlan 读 plan_course | 课程列表 |

> 核心点：**丁是门面，没有自己的数据生产** — 全部调乙/甲/丙 的接口做事。
