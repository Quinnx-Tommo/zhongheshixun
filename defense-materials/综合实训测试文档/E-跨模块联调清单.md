## 三、E2E 主流程演练（4 人一起跑）

> 这是综合实训**最终 demo**，4 人一起跑一遍，录屏作为验收证据。

### 3.1 主流程（10 分钟）

```
【开场 1min】
4 人一起上台，介绍项目：
"各位老师好，这是我们 4 人团队的综合实训项目 —— 四川省基层卫生人员网络培训平台。
我是甲，负责用户底座；我是乙，负责课程学习；我是丙，负责考试统计；我是丁，负责咨询小程序。
下面演示完整业务流程。"

【登录 1min】
甲：打开管理后台 5176 → admin/123456 登录 → 展示用户管理
乙：打开 web-student 5174 → student01/123456 登录 → 展示首页
丁：打开微信开发者工具 → student01/123456 登录 → 展示小程序首页

【课程 2min】
甲：管理后台 → 课程管理 → 添加课程"高血压诊疗规范"
乙：web-student → 课程中心 → 看到新课程 → 报名 → 进入学习
丁：小程序 → 课程列表 → 看到新课程 → 进入详情

【学习 2min】
乙：web-student → 学习页 → 播放视频 → 每 10s 上报进度 → 点"学完"
丁：小程序 → 学习页 → 播放视频 → 进度上报

【考试 2min】
丙：管理后台 → 试题管理 → 添加 10 题（3:5:2）→ 创建考试 → 自动组卷
乙：web-student → 考试中心 → 开始考试 → 答题 → 交卷 → 看成绩
丁：小程序 → 考试列表 → 开始考试 → 答题 → 交卷 → 看成绩

【统计 1min】
丙：管理后台 → 统计报表 → ECharts 三图 + 4 维度表

【咨询 1min】
丁：小程序 → 咨询 → 问"高血压怎么用药" → 命中知识库
丁：问刁钻问题 → 转人工
甲：管理后台 → 咨询管理 → 看到红色标红 → 回复
丁：小程序看到回复

【收尾 30s】
4 人一起："以上就是完整业务流程，谢谢！"
```
