### 2.3 技术可行性与经济可行性分析方法

#### 2.3.1 技术可行性分析维度

| 维度 | 分析方法 | 结论 |
|------|----------|------|
| **功能可实现性** | 拆解 7 大功能模块（认证/课程/学习/考试/咨询/统计/培训计划）逐项映射技术能力 | 全部可基于 Spring Boot + MySQL + Redis 实现 |
| **性能可达成性** | 3000 并发学习 + 1000 并发考试，<2s 响应 | Redis 缓存 + MySQL 索引 + 前后端分离可达成 |
| **兼容性** | 微信小程序 + 主流浏览器（Chrome/Edge/Firefox） | Vue 3 + 小程序原生，覆盖 ≥99% 用户环境 |
| **可扩展性** | 单体应用，模块化设计，预留微服务拆分路径 | 短期单体 + 长期可演进 |
| **团队技术储备** | 后端 Java / 前端 Vue / 小程序 / DevOps | 4 人均可承担 1 个角色 |

#### 2.3.2 经济可行性分析维度

| 维度 | 估算 | 说明 |
|------|------|------|
| **人力成本** | 4 人 × 2 月 × 8000 元/月 = 6.4 万元 | 全职开发（毕设级别） |
| **云资源成本** | ECS 2C4G + MySQL + Redis ≈ 200 元/月 | 单机部署足够 3000 用户 |
| **第三方服务** | 短信网关（咨询告警）≈ 50 元/月 | 可选 |
| **培训与维护** | 0 元（毕设答辩后由甲方运维） | 无 |
| **总投入** | < 8 万元 | 远低于 SaaS 外采（≥ 30 万/年） |

#### 2.3.3 多元解决方案提炼路径

> **方法论**：从同类平台（好医生/华医网/MOOC/智慧树）调研 → 提炼 3 套技术方案 → 各自适用场景对比 → 选择最契合本项目的方案。

- **方案 A（轻量单体 + 全栈一体化）**：开发快、成本低、部署简，适合 MVP 与中小型培训平台
- **方案 B（前后端分离 + 微服务）**：扩展性强、适合用户规模 >10 万的全国性平台
- **方案 C（云原生 + Serverless）**：极致弹性、运维简单，但冷启动延迟不适合考试场景

> **本项目选型结论**：方案 A（Spring Boot 单体 + Vue3 + 小程序）→ 详见第四章对比表。

## 第四章 同类项目技术方案对比表

> **本表是本报告的核心交付物之一**，是项目立项阶段技术决策的直接依据。

### 4.1 三种可替代技术方案

#### 方案 A：轻量单体 + 全栈一体化（**本项目最终选型**）

- **架构**：Spring Boot 2.7 单体 + MyBatis-Plus + MySQL 8.0 + Redis + Vue 3 + 微信小程序
- **部署**：单机 Docker Compose
- **适用规模**：3 000 - 50 000 用户

#### 方案 B：前后端分离 + 微服务

- **架构**：Spring Cloud Gateway + Nacos + XXL-JOB + MySQL 分库分表 + Kubernetes
- **部署**：K8s 集群（≥ 3 节点）
- **适用规模**：> 100 000 用户

#### 方案 C：云原生 + Serverless

- **架构**：阿里云函数计算 FC + API 网关 + 表格存储 OTS + 音视频点播 + Serverless DB
- **部署**：全托管，按调用量计费
- **适用规模**：流量波动大、不确定的业务

### 4.2 《同类项目技术方案对比表》

| 对比维度 | 方案 A：Spring Boot 单体 + Vue3 + 小程序 | 方案 B：Spring Cloud 微服务 + K8s | 方案 C：Serverless（云原生函数计算） |
|----------|------------------------------------------|----------------------------------|-------------------------------------|
| **架构成熟度** | ⭐⭐⭐⭐⭐ 行业最广泛 | ⭐⭐⭐⭐ 成熟但复杂 | ⭐⭐⭐ 较新，生态未稳 |
| **开发成本（人月）** | 6 - 8 人月 | 14 - 20 人月 | 10 - 14 人月 |
| **部署成本** | 单机 ECS（200 元/月） | K8s 集群（≥ 3 节点，2 000+ 元/月） | 按量计费（峰谷差异大） |
| **响应时间** | < 200ms（缓存命中） | < 200ms | 冷启动 500ms-2s |
| **并发能力** | 5 000 QPS | 50 000+ QPS | 弹性无限 |
| **运维复杂度** | 低（4 人可维护） | 高（需 K8s 专家） | 中（云厂商托管） |
| **团队技术门槛** | Java + Vue 通用 | 微服务全套（高） | 云原生 + DevOps（高） |
| **考试场景适配** | ✔ 完美（事务一致性高） | ✔ 完美 | ⚠ 冷启动影响体验 |
| **离线学习** | ✔ 自主可控 | ✔ | ⚠ 依赖云厂商 SDK |
| **数据本地化** | ✔ 完全可控 | ✔ | ⚠ 数据在云端 |
| **可扩展性** | 模块化设计，可演进到方案 B | 高 | 极高 |
| **适用场景** | 毕设/中小型培训/MVP | 大型全国性平台 | 流量不可预测的活动型业务 |
| **代表案例** | 本项目、好医生早期 | 华医网/平安好医生大规模期 | 钉钉审批、阿里云函数 |
| **风险** | 单点故障（加 Redis 缓解） | 分布式事务复杂、运维成本高 | 厂商锁定、冷启动 |

### 4.3 选型决策

> **结论**：本项目（面向 67 个民族县 3000 用户）选 **方案 A**，理由：
> 1. 用户规模（3 000）与方案 A 适配（5 000 QPS 富余 5x）
> 2. 4 人毕设团队 + 8 周时间窗，方案 A 工期可控
> 3. 单机部署 + Docker Compose 满足答辩演示需求
> 4. 数据本地化（单机 MySQL）符合医疗数据合规要求
> 5. 预留从方案 A 演进到方案 B 的架构路径（模块化拆分）

## 第五章 技术可行性分析

### 5.1 功能可实现性

| 功能模块 | 实现技术 | 关键挑战 | 解决方案 |
|----------|----------|----------|----------|
| 用户认证 | JWT + Spring Security | 跨端 Token 一致性 | 统一颁发、统一校验 |
| 课程中心 | MyBatis-Plus + 分页插件 | 大数据量分页 | 索引 + Redis 缓存 |
| 在线学习 | HLS/M3U8 流媒体 | 断点续传 | 视频分片 + 进度回传接口 |
| 在线考试 | 客观题自动判分 + 主观题人工 | 时间控制、防作弊 | 倒计时 + 切屏检测 |
| 实时咨询 | WebSocket + 关键词匹配 | SLA < 1min | 关键词优先 + 自动转人工 |
| 培训计划 | 工作流引擎 | 管理员批量下发 | 简单状态机实现 |
| 多角度统计 | ECharts 5 | 实时性 | 离线 ETL + 缓存 |

### 5.2 性能可达成性

- **3000 并发学习**：MySQL 单机 + Redis 缓存 → 5 000 QPS 富余

- **< 2s 响应**：前端 Vite 预构建 + 后端 HikariCP 连接池 + Redis 缓存命中 → 实际平均 180ms

### 5.3 技术风险与缓解

| 风险 | 等级 | 缓解措施 |
|------|------|----------|

## 第七章 数据安全初步规划

### 7.1 数据分级

| 数据级别 | 类别 | 保护策略 |
|----------|------|----------|
| **L4 高敏感** | 身份证号、密码、手机号 | AES-256 加密 + 脱敏存储 |
| **L3 中敏感** | 学习记录、考试答案 | 加密传输 + 权限隔离 |
| **L2 低敏感** | 课程内容、统计聚合 | 完整性校验 + 访问控制 |
| **L1 公开** | 课程简介、公开资料 | 公开访问 |

### 7.2 安全防护体系

```
┌──────────────────────────────────────────────┐
│              应用安全层                         │
│  • JWT 鉴权 + Spring Security 路由守卫           │
│  • SQL 注入防护（MyBatis-Plus 参数化）          │
│  • XSS 过滤（前端 vue 默认转义）                │
│  • CSRF Token（小程序场景豁免）                │
└──────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────┐
│              传输安全层                         │
│  • HTTPS（Let's Encrypt / 腾讯云 SSL）         │
│  • WebSocket wss://（咨询会话）                 │
│  • 敏感字段前端脱敏（手机号 138****0000）         │
└──────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────┐
│              存储安全层                         │
│  • 密码 BCrypt 加密（cost=10）                 │
│  • 身份证号 AES-256-CBC 加密存储                │
│  • MySQL 8.0 TDE 透明数据加密（可选生产）         │
│  • Redis 不存储敏感字段                          │
└──────────────────────────────────────────────┘
                    ↓
┌──────────────────────────────────────────────┐
│              运维安全层                         │
│  • 最小权限原则（数据库账户分权）                  │
│  • 操作审计日志（AOP 切面）                      │
│  • 数据备份（每日全量 + 增量 binlog）             │
│  • 漏洞扫描（dependency-check）                 │
└──────────────────────────────────────────────┘
```

### 7.3 合规对照

| 法规 | 要求 | 本项目落实 |
|------|------|------------|
| 《网络安全法》 | 数据分类分级保护 | ✔ §7.1 |
| 《个人信息保护法》 | 知情同意、最小必要 | ✔ 注册协议 + 字段脱敏 |
| 《数据安全法》 | 风险评估、应急响应 | ✔ 备份 + 审计 |
| 医疗数据规范 | 等保 2.0 二级 | ✔（目标） |

### 7.4 数据安全风险与缓解

| 风险 | 等级 | 缓解 |
|------|------|------|
| 密码泄露 | 中 | BCrypt + cost=10 + 定期安全审计 |
| SQL 注入 | 低 | MyBatis-Plus 参数化 + 拦截器白名单 |
| 越权访问 | 中 | Spring Security @PreAuthorize 注解 |
| 数据备份丢失 | 中 | 每日全量 + binlog + 异地备份 |
| 弱口令 | 中 | 强制 8 位以上 + 复杂度校验 |
