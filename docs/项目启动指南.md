```java
@Configuration

}
```

## 七、常见问题

### Q1: 数据库连接失败

**检查项**：
- MySQL 是否启动：`systemctl status mysql`
- 账号密码是否正确
- 数据库是否存在：`SHOW DATABASES;`

### Q2: Maven 依赖下载慢

**解决方案**：配置阿里云镜像
```xml
<!-- ~/.m2/settings.xml -->
<mirrors>
  <mirror>
    <id>aliyun</id>
    <mirrorOf>central</mirrorOf>
    <url>https://maven.aliyun.com/repository/public</url>
  </mirror>
</mirrors>
```

### Q3: 前端启动后无法访问后端

**检查项**：
- 后端是否正常启动（看控制台日志）
- 跨域配置是否添加
- 前端 API 基础地址是否正确

### Q4: 小程序接口 404

**检查项**：
- `training-api` 是否启动（端口 8081，若被占用可换 8082 并用 `--server.port=8082` 启动）
- 小程序 `baseURL` 配置是否正确
- 接口路径是否匹配设计文档

### Q5: JWT 认证失败

**检查项**：
- 请求头是否携带 `Authorization: Bearer <token>`
- Token 是否过期
- 密钥配置是否一致（8080 / 8081 / 8082 必须使用同一个 JWT secret）

### Q6: 示例数据看不到

**解决方案**：重新执行数据库初始化脚本
```bash
mysql -u root -p < docs/database.sql
```

### Q7: 登录失败（BCrypt 密码错误）

**已解决**：`docs/database.sql` 中账号密码已于 2026-07-10 统一更新为 `123456` 对应 hash。**所有演示账号统一密码为 `123456`**（admin / teacher01 / teacher02 / student01-06）。
**历史背景**：M10 测试发现 `database.sql` 中存储的 BCrypt hash 实际对应明文是 `123456`（不是 `password`），与文档说明的 `password` 不一致。已通过 ALTER + UPDATE 修复。

### Q8: 端口冲突（8081 / 8082 三端口混乱）

**说明**：
- **8080**：管理后台后端 + 统一认证中心（AuthController）
- **8081**：training-api 默认端口（**当前被僵尸进程占用，不可用**）
- **8082**：training-api 业务端口（web-student 代理目标）
- **5173**：Vue 管理后台前端
- **5174/5175**：web-student PC 学员端

如需切换端口：
```bash
# 8081 被占时，换用 8082
java -jar training-api/target/training-api-exec.jar --server.port=8082
```
