## 一、项目概述

### 1.1 项目背景
四川省67个民族县基层卫生人员网络培训平台，面向乡镇卫生院和社区卫生服务中心的七类卫生技术人员（临床医学、公共卫生、护理、医技B超/心电/检验/影像），共3000人，每人每年需完成200学时。

### 1.2 项目特点
- 课程期末作业项目
- 不需要高并发、安全、性能优化
- 仅用于功能演示和答辩验收

### 1.3 技术栈

| 模块 | 技术 | 说明 |
|------|------|------|
| **后端** | Spring Boot 2.7 + MyBatis-Plus | 单体应用 |
| **数据库** | MySQL 8.0 | 单机即可 |
| **缓存** | Redis（可选） | 作业场景可不使用 |
| **前端** | Vue3 + Element Plus | 后台管理 |
| **小程序** | 微信小程序原生 | 学员端 |
| **认证** | JWT | Token 认证 |

## 三、功能需求

### 3.1 角色权限（3种）

| 角色 | 权限 |
|------|------|
| **管理员** | 全部功能：用户管理、课程管理、考试管理、培训计划、统计报表、咨询管理 |
| **讲师** | 课程管理、试题管理、查看学员学习进度、在线答疑 |
| **学员** | 课程学习、在线考试、查看学习记录、咨询问答 |

### 3.2 核心功能模块

#### 模块1：用户管理
- 登录/注册（用户名密码 + JWT）
- 个人信息管理
- 角色权限控制
- 组织机构、岗位类型（临床/公卫/护理/医技）

#### 模块2：课程管理
- 课程 CRUD（标题、描述、封面、学时、类型）
- 章节管理（视频、时长、排序）
- 课程发布/下架
- 课程分类（公开课/必修课）
- **药柜式课件组合**：灵活选择课件组成不同课程

#### 模块3：学习功能
- 课程报名
- 视频播放
- 学习进度记录（章节进度、总进度、学习时长）
- 上次学习位置记忆（断点续播）

#### 模块4：考试模块（3种类型）

| 考试类型 | 说明 |
|----------|------|
| **课程考试** | 每门课程关联，学完课程后参加 |
| **计划考试** | 培训计划内的考试 |
| **单独考试** | 独立的结业考试 |

**考试功能：**
- 试题管理（单选、多选、判断、填空、问答）
- 知识点管理（知识点下归属试题）
- 自动组卷（按难度比例随机抽取：简单30%、普通50%、困难20%）
- 在线答题（计时）
- 自动阅卷（客观题自动判分）
- 考试记录查询
- 支持重考次数设置
- 支持及格分数设置

#### 模块5：培训计划
- 创建培训计划
- 关联多个课程（支持排序、必修/选修）
- 流程控制（按顺序学习）
- 计划发布/下架

#### 模块6：智能问答
- LongCat AI 自动回复：调用大模型生成智能应答
- 转人工关键词检测（命中"转人工/找老师"等 → 直接人工工单）
- AI 未启用或调用失败时兜底转人工
- 咨询记录留存

#### 模块7：统计报表
- 学员学习统计（学习时长、完成率）
- 考试成绩统计
- 课程热度统计
- 平台运行概览

#### 模块8：离线下载
- PC 端提供 ZIP 下载链接
- 包含视频文件和课程资料
- 简单直接，不加密（作业简化）

#### 模块9：培训讲师管理
- 讲师资源维护（教学方向、职称、学历）
- 讲师可进行课程管理
- 在线解答学员提问

## 六、核心算法

### 6.1 自动组卷算法

```java
public List<Question> generatePaper(Long examId, int count) {
    // 按难度比例随机抽取：简单30%、普通50%、困难20%
    int easy = count * 30 / 100;
    int medium = count * 50 / 100;
    int hard = count - easy - medium;

    List<Question> questions = new ArrayList<>();
    questions.addAll(questionMapper.randomByDifficulty(1, easy));
    questions.addAll(questionMapper.randomByDifficulty(2, medium));
    questions.addAll(questionMapper.randomByDifficulty(3, hard));

    Collections.shuffle(questions);
    return questions;
}
```

### 6.2 智能问答算法

```java
public ConsultResult askQuestion(String question) {
    // 1. 检测转人工关键词（命中"转人工/找老师"等 → 直接转人工）
    if (containsTransferKeyword(question)) {
        return ConsultResult.transfer("正在为您转接人工客服...");
    }

    // 2. 调用 LongCat AI 自动回复（启用时）
    String aiReply = longCatAiService.ask(question);
    if (StringUtils.hasText(aiReply)) {
        return ConsultResult.success(aiReply);
    }

    // 3. AI 未启用或调用失败 → 兜底转人工
    return ConsultResult.transfer("正在为您转接人工客服...");
}
```

### 11.2 高并发技术方案（docx 第 4 节 — 三类瓶颈 + 技术选型）

docx 明确指出高并发来源于**传输层、计算层、存储层**。以下为完整技术方案（答辩需掌握）：

#### 传输层（用户请求 → Web 服务器）

| 瓶颈 | docx 要求的技术 | 作用 | 作业实现方式 |
|------|----------------|------|-------------|
| 网络带宽 + I/O | **CDN** | 用户访问最近缓存节点 | Redis 模拟热点课程缓存（伪代码） |
| 跨网访问 | **智能双路** | 按运营商智能切换线路 | 设计说明（需多线机房部署） |
| Web 服务器压力 | **LVS** | 四层负载均衡，压力分发 | Nginx 七层 upstream 负载均衡 |
| Web 反向代理 | **HA-Proxy** | 请求分发到多台 Web 服务器 | Nginx `upstream` 配置 |
| 资源消耗大户 | **流媒体服务器分离** | 视频独立，避免页面服务器崩溃 | 视频独立 `/video/` + 独立 Nginx location（docx 强调） |
| 低频更新内容 | **HTML 静态化** | 避免频繁 DB 访问 | Thymeleaf 模板缓存（资讯页） |

**流媒体分离架构**（docx 重点强调，见 `开发文档.md` 第十五章）：
```
用户请求 ──→ Nginx
               ├─ /api/*  → Spring Boot（业务接口）
               ├─ /video/*→ 流媒体目录 /home/training/video/（独立存储）
               └─ /*      → Vue3 静态资源
```

#### 计算层（业务处理）

| 瓶颈 | docx 要求的技术 | 作用 | 作业实现方式 |
|------|----------------|------|-------------|
| 海量数据并行处理 | **MapReduce** | 分布式计算、分载压力 | Java `parallelStream()` + 线程池模拟 Map→Reduce |
| 并行计算模型 | **BSP 模型** | 大规模同步并行 | `ForkJoinPool` 任务拆分聚合 |
| 缓解存储压力 | **Result Cache** | 计算结果缓存 | Redis 缓存统计结果（带 TTL） |
| 内部并行分发 | **Scatter/Gather** | 内部服务器并行计算后聚合 | `CompletableFuture` 并发查询后汇总 |

**MapReduce 概念验证代码**（详见 `开发文档.md` 第十五章）：
```java
// 模拟 MapReduce：学员学习统计任务分发到多线程
// Map：每个线程计算一个分区
List<CompletableFuture<Map<String, Object>>> futures = partitions.stream()
    .map(p -> CompletableFuture.supplyAsync(() -> statsService.compute(p), pool))
    .collect(toList());
// Reduce：汇总
Map<String, Object> result = futures.stream()
    .map(CompletableFuture::join)
    .reduce(new HashMap<>(), StatsUtils::merge);
```

#### 存储层（数据库）

| 瓶颈 | docx 要求的技术 | 作用 | 作业实现方式 |
|------|----------------|------|-------------|
| 读远大于写 | **读写分离** | 1主多从，分散读请求 | 双数据源 demo（`AbstractRoutingDataSource`） |
| 时间段查询 | **分区策略** | 按时间提高读写性能 | MySQL `PARTITION BY RANGE` |
| 单库并发瓶颈 | **Sharding（库表散列）** | 数据分散存储 | user_id % 4 分表 demo |
| 高可用 | **数据库集群与镜像** | 虚拟单一 DB 逻辑映像 | 主从复制（1主1从） |
| 降低开销 | **只镜像关键数据** | 频繁复制降低效率 | 仅镜像日志表（docx 原文） |
| 海量查询 | **Column-Based** | 列式存储提高查询效率 | MySQL 索引优化讲解 |

**读写分离概念验证代码**：
```java
public class RoutingDataSource extends AbstractRoutingDataSource {
    @Override
    protected Object determineCurrentLookupKey() {
        return TransactionContext.isReadOnly() ? "slave" : "master";
    }
}
// @Transactional(readOnly = true) 走 slave；写操作走 master
```
