# 2026-07-08-专业文档增量填写策略.md

## 策略：边开发边填写，不单独批量完成

### 规则

每完成一个项目产出（代码/设计/测试），**顺手更新对应专业文档**。

### 可自动化的部分（可在编码完成时同步写）

| 项目产出 | 对应文档 | 填写位置 | 提示 |
|---------|---------|---------|------|
| `database.sql` 表结构定稿 | 4.数据库设计说明书 | 第3章 数据库表设计 | 用脚本自动提取字段清单 |
| 模块分层（common/dao/service/admin/api） | 5.系统架构设计说明书 | 第2章 架构视图 | 画分层图 |
| 项目搭建完成 | 3.项目开发计划 | 第3章 工作分解 | 时间线、里程碑 |
| 接口清单（dev-api.md） | 2.软件需求规约 / 7.测试用例 | 功能清单 / 测试场景 | 自动生成 |
| 阶段测试通过 | 8.测试日志 | 测试记录 | 截图填充 |

### 无法自动化、须学生补充的部分

| 文档 | 必须手动填写 |
|------|------------|
| 1.配置管理计划书 | 组织、CCB、基线定义 |
| 9.项目例会纪要 | 实际会议记录 |
| 10.项目开发总结完成时 | 你的心得体会 |
| 11.项目问题跟踪表 | 开发时的真实问题 |

### 注意事项

1. 模板来自其他项目（智慧教育研教部），大部分内容需替换为本项目的实际内容
2. 同学 Ga 在 7/1 已编辑过 2.软件需求规约，修改前与 Ga 确认
3. 文档增量填写时保留模板编号/表格格式
4. 建议填写格式：我的项目产出 → Markdown 草稿 → 学生复制到 Word

### 最后提醒

毕业答辩时，这些文档由学生统一提交给老师，作为毕设评分的重要依据。
