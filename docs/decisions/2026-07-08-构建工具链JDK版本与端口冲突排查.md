## 1. 【已解决】Lombok 初始化失败：ExceptionInInitializerError

### 现象
```
java: java.lang.ExceptionInInitializerError
com.sun.tools.javac.code.TypeTag :: UNKNOWN
```

### 根因
- 项目要求 **JDK 17**（`JAVA_HOME=D:\javaEE\soft\jdk-17.0.9`）
- 但 Windows 系统 PATH 里 **JDK 8**（`C:\Program Files (x86)\Common Files\Oracle\Java\java8path`）排在前面
- shell 实际 `java -version` 显示 `1.8.0_431`，`mvn` 在 JDK 8 上跑
- Lombok 1.18.30 使用 javac 内部 API，在 JDK 8 上触发 `TypeTag` 枚举初始化失败

### 修复
**不要改系统 PATH**（用户决定自己在项目里改）。临时修复：
```bash
export JAVA_HOME="D:/javaEE/soft/jdk-17.0.9"
export PATH="$JAVA_HOME/bin:$PATH"
mvn -q -DskipTests clean package
```

### 启动 jar 必须用 JDK 17 绝对路径
`export PATH` 在 Git Bash 不真正生效（被 JDK 8 覆盖），**必须用绝对路径**：
```bash
JAVAD="D:/javaEE/soft/jdk-17.0.9/bin"
nohup "$JAVAD/java" -jar training-api/target/training-api-exec.jar > logs/api_boot.log 2>&1 &
nohup "$JAVAD/java" -jar training-admin/target/training-admin-exec.jar > logs/admin_boot.log 2>&1 &
```

### 验证
`"$JAVAD/java" -version` 输出 `java version "17.0.9"` 即成功。

## 2. 【已解决】Port 8081 already in use

### 现象
```
PortInUseException: Port 8081 is already in use
```

### 根因
上次 session 用 nohup 启动的 jar 进程未退出，一直占用端口。

### 修复
```bash
# 查占用
netstat -ano | grep :8081 | grep LISTENING
# 显示 PID
wmic process where "processid=<PID>" get commandline
# kill
taskkill //F //PID <PID>
```

### 预防
- 用 nohup 手动启动测试后，**关 IntelliJ 前先 kill nohup 进程**
- IntelliJ 启动前先 `netstat -ano | grep :8081` 检查

## 3. 【已解决】UnsupportedClassVersionError（class 61.0 vs 52.0）

### 现象
```
java.lang.UnsupportedClassVersionError: com/training/api/ApiApplication has been compiled by a more recent version of the Java Runtime (class file version 61.0), this version of the Java Runtime only recognizes class file versions up to 52.0
```

### 根因
jar 用 JDK 17 编译（class 61.0），但跑在 JDK 8 上（只支持到 52.0）。
= JDK 8 启动 jar 的另一个表现（与问题 1 同源）。

### 修复
同问题 1：用 JDK 17 绝对路径启动。

## 6. 【已解决】WXML 编译错误：wx:elif 缺值

### 现象
```
[ WXML 文件编译错误] ./pages/consult/index/index.wxml
Bad attr `wx:elif` with message: value not set.
  44 |       <view class="row answer-row" wx:elif>
```

### 根因
小程序咨询页 wxml 里 `wx:elif` 没给条件值。

### 修复
`miniprogram/pages/consult/index/index.wxml`：
```xml
<!-- 旧：<view class="row answer-row" wx:elif> -->
<view class="row answer-row" wx:elif="{{!item.answer}}">
```
