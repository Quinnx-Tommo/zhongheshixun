# 2026-07-08 小程序修复 + 真实后端对接 + 课程类型语义统一

> 创建时间：2026-07-08
> 决策者：Claude + 学生
> 状态：已落地

## 背景

小程序端存在 4 个 bug（"我的"页缺失、启动强制跳登录、首页入口链接错、后端 profile 接口缺失），且 course/exam 两个模块的 API 层仍使用 mock 数据，未与数据库联通。同时课程类型字段 `course_type` 在小程序端存在语义错乱（数据库 `1=公开课/2=必修课`，前端写反）。

## 决策

### 1. 小程序 bug 修复（全部落地）

| 问题 | 修复方案 |
|------|---------|
| tabBar "我的" 指向 `pages/login/login` | 新建 `pages/profile/profile` 4 个文件，tabBar 改指向新页面 |
| `app.js` onLaunch 强制跳登录 | 移除强制跳转，由各 tabBar 页 onShow 各自检查登录态 |
| 首页"考试/计划/我的"入口指向登录页 | 改为真实页面路径（`/exam/list/list`、`/plan/plan`、`/profile/profile`） |
| 后端无 `/api/user/profile` | 在 `WxAuthController` 新增 GET/PUT `/api/user/profile` |

### 2. mock → 真实后端（全部落地）

- `api/course.js`：全部替换为 `request()` 调用，含 `getList/getMyCourses/getDetail/getChapters/enroll/reportProgress/getProgress`
- `api/exam.js`：全部替换为 `request()` 调用，含 `getList/getMyRecords/startExam/submitExam/getRecord`
- 新增字段映射：后端 `courseType 1/2` → 前端 `courseTypeText '公开课'/'必修课'`；后端 `PageResult` → 前端 `{ records, total, hasMore }`
- JSON 选项字符串（`["A","B"]`）在前端自动 parse 为数组

### 3. 课程类型语义统一（全部落地）

数据库定义 `1=公开课 / 2=必修课`，但小程序 `course/list/list.wxml` 和 `plan/plan.wxml` 写反了。统一修复：

- `pages/course/list/list.js`：筛选器改为 `必修=2 / 公开=1`
- `pages/course/list/list.wxml`：标签 `courseType===2 ? '必修课' : '公开课'`
- `pages/course/detail/detail.wxml`：同上
- `pages/plan/plan.wxml`：标签 `courseType===2 ? '必修' : '选修'`

### 4. 页面联动增强

- `course/list/list.js`：支持 `?mode=mine` 切换全部课程 / 我的课程
- `course/detail/detail.js`：报名按钮接真实接口 + 学习进度查询
- `exam/list/list.js`：并发拉考试 + 用户记录，合并字段打标（已考/待考/得分）
- `exam/result/result.js`：支持从 recordId 拉详情
- `profile/profile.js`：学习数据（课程数/考试数/咨询数/学习时长）并发请求真实接口
- `plan/plan.js`：学习计划数据从真实课程/考试接口拉取

## 影响

- 小程序端数据链路全部打通：**前端 → 8081 后端 → MySQL**
- 数据库已初始化 4 门课程（3 门已发布）、3 场考试、若干试题
- 后端编译通过（`mvn compile` BUILD SUCCESS）
- 小程序 WXML 编译错误已修复（plan/plan.wxml 少 `}`）

## 风险

- 8081 端口在新对话中被僵尸进程死锁（Windows 内核级），已改 8082（见进度文档 §10.3）
- 小程序 `WxAuthController.wxLogin` 是模拟 openid（毕设简化），无法用于 Web 端登录 → Web 端复用 8080 `/admin/login`
