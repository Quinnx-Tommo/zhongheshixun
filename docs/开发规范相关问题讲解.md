## 一、PC端和移动端统一接口问题

PC端管理后台

移动端（APP，小程序）

             PC端
              |
              |
          t_Setmeal接口
              |
              |
            后端
              |
              |
          移动端

为每一个端开发一个接口就会导致代码的冗余，接口重复，后期维护困难

因此一般采用：**一个Controller接口同时支持多个客户端调用**。

### 如何区分不同客户端的请求

#### 方法一：通过User-Agent判断

HTTP请求头中有 `User-Agent`，例如：

- PC：`Mozilla/5.0 Windows`

- 移动端：`Mozilla/5.0 Android`

- 后端可以通过解析：

  ```java
  String userAgent = request.getHeader("User-Agent");
  ```


#### 方法二：自定义请求头（推荐）

移动端：

header:
client-type: mobile

PC端：

header:
client-type: pc

后端：

String clientType = request.getHeader("client-type");

if("mobile".equals(clientType)){
    //移动端逻辑
}else{
    //PC端逻辑
}

## 三、统一token身份认证

传统登录，登录成功以后，浏览器存session 

前后端分离，多端访问，分布式部署问题

token进行身份认证

### 2. Token流程

**登录流程：**

```
用户输入账号密码
        ↓
后端验证
        ↓
生成Token
        ↓
返回客户端
```

例如：

{
  "token":"eyJhbGciOiJIUz..."
}

#### 3.请求的时候请求头header携带token

后端通过拦截器验证token

请求
 ↓
TokenInterceptor
 ↓
验证token
 ↓
Controller
