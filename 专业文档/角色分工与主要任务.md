# 角色分工与主要任务

## 一、项目概述

本项目为**四川省基层卫生人员网络培训平台**，是课程期末作业项目，面向四川省 67 个民族县 3000 名基层卫生技术人员，提供 MOOC 学习、在线考试、离线学习、实时咨询等服务。

| 项目属性 | 内容 |
|---------|------|
| **团队规模** | 4 人 |
| **开发周期** | 1-2 周（实际 9.2 天） |
| **项目性质** | 课程期末作业 |
| **原始需求** | `11.基层卫生人员网络培训平台.docx` |
| **版本控制** | Git |

```

### 5.2 分支策略（Git Flow 简化版）

| 分支 | 用途 | 规则 |
|------|------|------|
| `main` | 稳定版本，可部署 | 仅接受 `develop` 分支合并，禁止直接提交 |
| `develop` | 开发版本，日常集成 | 成员日常开发的基础分支 |
| `feature/xxx` | 功能分支 | 从 `develop` 检出，完成后合并回 `develop` |
| `hotfix/xxx` | 紧急修复 | 从 `main` 检出，修复后合并到 `main` 和 `develop` |

**分支协作流程**：

```
main  ──────────────────────────────────────────────
        ↑                                          ↑
       merge                                      merge
        │                                          │
develop ─┬────────┬────────┬────────────────────────
         │        │        │
      checkout  checkout checkout
         │        │        │
feature/login  feature/exam  feature/stats
```

### 5.3 提交规范

| 前缀 | 用途 | 示例 |
|------|------|------|
| `feat:` | 新功能 | `feat: 实现自动组卷算法` |
| `fix:` | Bug 修复 | `fix: 修复考试分页失效问题` |
| `docs:` | 文档更新 | `docs: 更新 API 接口文档` |
| `refactor:` | 重构 | `refactor: 优化 StatsService 查询逻辑` |
| `chore:` | 构建/工具 | `chore: 更新 Maven 依赖版本` |

### 5.4 .gitignore 关键规则

```
# Java / Maven 构建产物
target/
*.class
*.jar
*.war

# Node / Vue 依赖与构建产物
node_modules/
dist/

# IDE 配置（避免个人配置冲突）
.idea/
.vscode/
*.iml

# 环境配置 / 敏感信息（安全合规）

## 八、分工总结

本项目采用**"接口先行、并行开发、分阶段联调"**的协作模式：

1. **成员A 作为后端架构师**，承担项目基础设施搭建（Maven 多模块、数据库、JWT 认证），并输出接口文档供前端和小程序并行开发
2. **成员B 专注后端业务模块**，在成员A 搭建的框架上实现考试、统计、咨询等复杂业务逻辑
3. **成员C 负责 PC 双端前端**（管理后台 + 学员网页端），按接口文档对接后端
4. **成员D 负责移动端小程序**，与成员C 共用同一套后端 API，保证多端数据一致性

通过 Git 版本控制 + 5 个 Sprint 的开发计划 + 明确的接口契约，4 人团队在 1-2 周内完成了从需求分析到代码实现的完整闭环。
