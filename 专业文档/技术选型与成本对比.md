# 技术选型与成本对比

## 一、选型原则

本项目为**课程期末作业**，面向四川省基层卫生人员网络培训平台（67 个民族县，3000 名基层卫生技术人员）。在选型过程中遵循以下原则：

| 原则 | 说明 |
|------|------|
| **优先开源** | 全部核心组件选用开源方案，零授权费用，降低项目投入 |
| **社区活跃** | 选择社区活跃、文档完善、生态成熟的技术，降低学习成本和维护风险 |
| **轻量易用** | 优先选择配置简洁、开箱即用的框架，适配 1-2 周的开发周期 |
| **国产兼容** | 考虑国产操作系统和国产化替代趋势，选用国内广泛使用的技术栈 |
| **生态一致** | 前后端技术栈在各自领域保持统一，避免技术碎片 |

## 三、分层技术选型对比

### 3.1 后端开发框架

| 对比维度 | **开源方案：Spring Boot 2.7.18**（采用） | 商业方案：Oracle WebLogic 14c |
|---------|--------------------------------------|---------------------------|
| **授权费用** | 免费（Apache 2.0） | Oracle WebLogic SE: **$5,000/CPU/年**；EE: **$10,000/CPU/年** |
| **开发成本** | 低，开箱即用，配置极简 | 高，需购买 License、配置域、部署EAR/WAR |
| **学习曲线** | 平缓，社区文档丰富 | 陡峭，需掌握 WebLogic 专有概念 |
| **社区生态** | 全球最大 Java 生态，GitHub Stars 72k+ | Oracle 官方维护，社区较小 |
| **启动速度** | 快（内嵌 Undertow/Tomcat） | 慢（需启动 Admin Server + Managed Server） |
| **微服务支持** | Spring Cloud 原生支持 | 需额外购买 Oracle Coherence |
| **适用场景** | 中小型项目、敏捷开发、教学 | 大型企业级、金融级高可用 |

**成本节省**：采用 Spring Boot 替代 WebLogic，每年节省授权费约 **$5,000~10,000/CPU**（折合 **¥35,000~70,000/CPU**）。本项目使用单机部署，相当于节省约 **¥3.5~7 万元/年**。

---

### 3.2 ORM 框架

| 对比维度 | **开源方案：MyBatis-Plus 3.5.3**（采用） | 开源方案：Hibernate 6.x |
|---------|--------------------------------------|----------------------|
| **授权费用** | 免费（Apache 2.0） | 免费（LGPL） |
| **代码生成** | 内置代码生成器（Entity/Mapper/Service/Controller） | 无原生生成器，需借助第三方 |
| **分页插件** | 内置分页插件，一行配置 | 需手写 Pageable 或引入 Spring Data |
| **动态 SQL** | XML + Lambda 条件构造器 | HQL / JPQL / Criteria API |
| **学习成本** | 低，SQL 可控，适合教学 | 中，需理解 JPA 规范 |
| **性能** | 直接映射 SQL，无额外开销 | 一级/二级缓存，但存在 N+1 问题 |

> 注：ORM 层无论选择 MyBatis-Plus 还是 Hibernate，均为开源免费方案，不涉及商业授权成本。选择 MyBatis-Plus 的主要原因是**代码生成器**和**分页插件**开箱即用，大幅降低 CRUD 开发工作量，适合 1-2 周的短周期项目。

### 3.3 关系型数据库

| 对比维度 | **开源方案：MySQL 8.0**（采用） | 商业方案：Oracle 19c |
|---------|------------------------------|-------------------|
| **授权费用** | 免费（Community Edition，GPL） | Standard Edition 2: **$17,500/CPU**；Enterprise: **$47,500/CPU** |
| **运维成本** | 低，社区资料丰富 | 高，需 DBA 专业认证 |
| **高可用方案** | MySQL Group Replication / MGR（免费） | Oracle RAC（需额外 License） |
| **性能监控** | Percona / PMM（免费） | Oracle Enterprise Manager（收费） |
| **分区表** | 支持（本项目采用 RANGE 分区） | 支持（功能更完善） |
| **存储引擎** | InnoDB（默认，支持事务/行锁） | 单一引擎，功能最强 |
| **云厂商支持** | 阿里云 RDS MySQL、腾讯云 CDB（¥100+/月） | 阿里云 RDS Oracle（¥2,000+/月） |

**成本节省**：采用 MySQL Community Edition 替代 Oracle，授权费节省约 **$17,500~47,500/CPU**（折合 **¥12.5~34 万元/CPU**）。云部署场景下，阿里云 MySQL RDS 起步价约 **¥100/月**，同规格 Oracle RDS 约 **¥2,000+/月**，月度成本降低 **95%**。

| 对比维度 | **开源方案：Redis 7.x**（采用） | 商业方案：Aerospike / GridGain |

**成本节省**：Redis 单机部署零成本。若采用 Aerospike EE，以 16GB 缓存为例，年费用约 **$96**（¥700），但对于课程作业项目，Redis 完全满足需求。

## 五、选型决策说明

### 5.1 为什么优先选择开源方案？

| 因素 | 说明 |
|------|------|
| **项目性质** | 课程期末作业，非商业生产系统，无预算购买商业授权 |
| **开发周期** | 仅 1-2 周，商业软件的安装、配置、License 申请会占用大量时间 |
| **学习价值** | 开源技术（Spring Boot / Vue3 / MySQL）是业界主流，学习价值高于封闭商业产品 |
| **部署灵活** | 开源方案可在个人电脑、学校机房、云服务器任意环境部署，不受 License 绑定 |
| **社区支持** | 遇到问题可通过 Stack Overflow、GitHub Issues、中文社区快速获取帮助 |
| **未来扩展** | 开源技术栈与云原生生态（Docker / Kubernetes / 微服务）无缝衔接 |

### 5.2 开源方案是否满足项目需求？

| 需求项 | 项目要求 | 开源方案能力 | 是否满足 |
|--------|---------|-----------|---------|
| 并发用户 | 3000 日常并发学习，1000 并发考试 | Spring Boot + MySQL 单机可支撑 5000+ QPS | ✅ 满足 |
| 响应时间 | < 2 秒 | Spring Boot + Redis 缓存，平均响应 < 200ms | ✅ 满足 |
| 数据安全 | 医疗数据隐私保护 | Spring Security + BCrypt + JWT，符合等保二级 | ✅ 满足 |
| 离线学习 | 断点续传 + 进度回传 | 基于 offline_flag 标记 + zip 包下载，纯业务实现 | ✅ 满足 |
| 弱网优化 | 基层网络环境适配 | CDN 缓存预热 + 资源压缩 + 图片懒加载 | ✅ 满足 |
| 多语言支持 | 少数民族语言切换准备 | Vue i18n + 微信小程序国际化，架构预留 | ✅ 满足 |

### 5.3 商业方案何时需要引入？

| 场景 | 当前方案 | 未来升级路径 |
|------|---------|-------------|
| 用户规模 > 10 万 | Spring Boot 单体 | Spring Cloud 微服务 + Kubernetes |
| 数据库 QPS > 10 万 | MySQL 单机 | MySQL 主从 + 分库分表（ShardingSphere，开源） |
| 缓存容量 > 100GB | Redis 单机 | Redis Cluster（开源） |
| 金融级数据安全 | BCrypt + JWT | 国密 SM2/SM3/SM4（开源库支持） |
| 企业级 SSO | 自建 JWT | Keycloak / CAS（开源 IAM） |

> 即使项目规模扩大，开源生态仍提供了完整的升级路径（Spring Cloud、ShardingSphere、Redis Cluster、Keycloak 等），无需转向商业产品即可支撑百万级用户。

## 七、总结

本项目从后端框架、数据库、缓存、前端到基础设施，**全部选用开源/免费方案**，在满足 3000 用户并发、响应时间 < 2 秒、数据安全合规等项目需求的前提下，实现了**零软件授权成本**。

与采用商业方案（WebLogic + Oracle + F5 + 商业 UI 库 + 商业 IAM）相比，本项目在单服务器场景下**每年可节省约 ¥34 万元**的软件授权费用。即使未来项目规模扩大，开源生态（Spring Cloud、ShardingSphere、Redis Cluster、Kubernetes 等）仍提供了完整的免费升级路径，无需引入商业产品即可支撑百万级用户。

对于课程作业项目而言，开源方案不仅是成本最优解，更是**技术学习价值最高**的选择——Spring Boot、Vue3、MySQL、Redis 均为当前业界主流技术栈，掌握这些技术对未来的职业发展具有直接帮助。
