# 框架搭建与整合

## 一、技术栈概览

本项目基于 Spring Boot 2.7.18 单体架构，采用 MyBatis-Plus 3.5.3 作为 ORM 框架，MySQL 8.0 作为数据库，Redis 7.x 作为缓存，JWT (jjwt 0.11.5) 实现无状态认证。Maven 多模块分层组织，共 5 个子模块：

| 模块 | 职责 |
|------|------|
| `training-common` | 公共层：DTO、VO、统一响应、异常、常量、工具类 |
| `training-dao` | 数据访问层：Entity、Mapper 接口 |
| `training-service` | 业务逻辑层：Service 接口与实现 |
| `training-admin` | 后台管理后端（端口 9898）+ Spring Security RBAC |
| `training-api` | 小程序 API 后端（端口 9899）+ JWT 拦截器 |

## 三、数据源配置

### 3.1 HikariCP 连接池

两个后端模块（admin / api）共享同一数据库 `training`，均使用 Spring Boot 默认的 HikariCP 连接池：

```yaml
# training-admin/src/main/resources/application.yml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/training?useUnicode=true&characterEncoding=utf-8&serverTimezone=Asia/Shanghai&useSSL=false&allowPublicKeyRetrieval=true
    username: root
    password: root
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      maximum-pool-size: 20       # 最大连接数
      minimum-idle: 5             # 最小空闲连接
      connection-timeout: 30000   # 连接超时 30s
      idle-timeout: 600000        # 空闲超时 10min
      max-lifetime: 1800000       # 连接最大生命周期 30min
```

> `training-api` 的数据源配置与 admin 相同，仅 Redis database 编号不同（admin=0, api=1），实现缓存隔离。

### 3.2 Redis 缓存配置

```yaml
spring:
  redis:
    host: localhost
    port: 6379
    password:
    database: 0           # admin 用 db0，api 用 db1
    timeout: 5000
    lettuce:
      pool:
        max-active: 20
        max-idle: 10
        min-idle: 2
```

```yaml
mybatis-plus:
  mapper-locations: classpath*:mapper/**/*.xml
  type-aliases-package: com.training.common.entity
  configuration:
    map-underscore-to-camel-case: true   # 下划线转驼峰

      id-type: auto                       # 主键自增
      logic-delete-field: deleted         # 逻辑删除字段
      logic-delete-value: 1
      logic-not-delete-value: 0
```

### 4.2 分页插件配置

```java
// training-admin/src/main/java/com/training/admin/config/MybatisPlusConfig.java
@Configuration
public class MybatisPlusConfig {

    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        MybatisPlusInterceptor interceptor = new MybatisPlusInterceptor();
        interceptor.addInnerInterceptor(new PaginationInnerInterceptor(DbType.MYSQL));
        return interceptor;
    }
}
```

### 4.3 Mapper 接口映射（数据访问层）

所有 Mapper 继承 `BaseMapper<T>`，通过 `@Mapper` 注解注册，由 `@MapperScan("com.training.mapper")` 扫描：

```java
// 用户 Mapper — 含自定义关联查询
@Mapper
public interface SysUserMapper extends BaseMapper<SysUser> {
    SysUser selectByUsername(@Param("username") String username);
    SysUser selectUserById(@Param("id") Long id);
    IPage<SysUser> selectUserPage(IPage<SysUser> page,
                                  @Param("role") String role,
                                  @Param("status") Integer status,
                                  @Param("keyword") String keyword);
}

// 角色 Mapper
@Mapper
public interface SysRoleMapper extends BaseMapper<SysRole> {
    SysRole selectByCode(@Param("roleCode") String roleCode);
}

// 权限 Mapper
@Mapper
public interface SysPermissionMapper extends BaseMapper<SysPermission> {
    List<SysPermission> selectByRoleId(@Param("roleId") Long roleId);
}
```

## 五、事务管理

### 5.1 声明式事务

Spring Boot 自动配置 `DataSourceTransactionManager`，业务层通过 `@Transactional` 注解声明事务边界：

```java
// 考试业务 — 交卷时保证答题记录 + 试卷状态原子性
@Transactional(rollbackFor = Exception.class)
public void submitExam(...) { ... }

// 咨询业务 — 接单/回复/闭环保证数据一致性
@Transactional(rollbackFor = Exception.class)
public void acceptConsult(...) { ... }

// 培训计划 — 删除计划同时清理关联课程
@Transactional(rollbackFor = Exception.class)
public void deletePlan(...) { ... }

// RBAC 数据初始化 — 角色 + 权限 + 关联批量插入
@Transactional(rollbackFor = Exception.class)
@PostConstruct
public void init() { ... }
```

### 5.2 事务使用统计

项目中共 8 处 `@Transactional` 注解，分布在 4 个业务类中：

| 类 | 方法数 | 场景 |
|----|--------|------|
| `ExamBizServiceImpl` | 3 | 交卷、自动评分、批量操作 |
| `ConsultServiceImpl` | 3 | 接单、回复、闭环 |
| `TrainPlanServiceImpl` | 1 | 删除计划 + 清理关联 |
| `RbacDataInitializer` | 1 | 启动时初始化 RBAC 数据 |

所有事务均指定 `rollbackFor = Exception.class`，确保受检异常也触发回滚。
